# 耳边 MVP 开发任务清单

## 1. 使用说明
本清单基于《耳边 MVP 产品需求文档（PRD）》与《耳边 MVP 开发拆解文档》整理，目标是把需求转成可以直接执行、排期、挂 GitHub issue 的任务列表。

建议使用方式：
- P0 先做，确保 MVP 能跑通；
- P1 在 P0 稳定后补齐；
- P2 作为增强项；
- 每个任务都应进一步拆成 issue / subtask。

---

## 6. 建议 GitHub issue 拆分方式
每个任务建议采用以下模板：
- 标题：`[模块] 动作 + 结果`
- 示例：`[内容] 完成内容详情页接口`

建议字段：
- 背景
- 目标
- 范围
- 验收标准
- 依赖项
- 风险点

建议标签：
- `product`
- `frontend`
- `backend`
- `ai`
- `admin`
- `ops`
- `p0`
- `p1`
- `p2`

## 8. 验收总清单
MVP 至少满足：
- 用户可注册登录
- 用户可浏览内容并试听
- 用户可购买单条内容
- 用户可购买 VIP
- 用户可购买 AI 时长包
- 用户可与 AI 角色完成基础语音对话
- 用户可查看会员和资产状态
- 主播可上传内容并提交审核
- 平台可审核并上架内容
- 订单支付成功后权益立即生效

---

## 9. 结论
如果要快速推进，最优执行方式是：
1. 先按本清单建立 GitHub Project 和 issues；
2. 再按 P0 分配到前端、后端、设计、AI；
3. 每周只看 3 个核心结果：能不能听、能不能聊、能不能付费。
